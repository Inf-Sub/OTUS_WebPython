Что важно помнить про HTTP:

  • Вся работа происходит поверх HTTP
  • HTTP - текстовый протокол
  • С помощью текста загружается всё - и картнки, и HTML, и CSS, и JS
  • В HTTP 1.1 - на каждый файл делается отдельный запрос к серверу

Что важно помнить про HTTP:

  • HTTP строится на «запрос-ответ»
  • И Web-сокеты, если что - тоже
  • Есть ряд HTTP-методов (глаголов) для запросов
  • Есть ряд статус-кодов для респонзов

Упражнение: Какие HTTP-методы Вы знаете?

GET получение страниц получение entity (без тела)
POST отправка данных формы создание entity
PUT изменение entity
PATCH изменение entity (RFC-...)
DELETE удаление entity
HEAD, OPTIONS хитрые
TRACE не используется

  • 2xx - всё OK

  • 4xx - ошибка, виноват клиент

Что важно помнить про HTML:

  • HTML - прежде всего разметка (данные + структура)
  • Вариантов написания HTML много
  • XHTML - строгий стандарт со всеми закрывающимися тегам
  • HTML 5 - новый стандарт
  • HTML 5 - не только новые теги, это ещё и Local- и Session Storage, DOM API и многое другое

Что важно помнить про HTML:

  • CSS, JS, картнки, загружаются (и выполняются) в том порядке, как встречаются
  • Ссылки на CSS ставятся в начало страницы
  • JS - в конце
  • В современных приложениях разметку генерирует JS

Упражнение: Какие HTML-теги Вы знаете и для чего они?

Упражнение: Какие HTML-теги Вы знаете и для чего они?

  • <p> - это абзац (много текста!)
  • <br/> - перевод строк в абзаце! (и нигде больше)
  • <span> - это мало текста (меньше, чем одна строка)
  • <div> - блок
  • <nav>, <article>, <main> - те же div, только для SEO
  • <a> - это ссылка
  • <input type="button">, <button> - это кнопка

Что необходимо знать про JS?

  • ECMA Script - стандарт, JavaScript, по сути реализация
  • То что сейчас исполняется в браузере - ECMA Script 5.1 (ES 5.1)
  • Сравнительно вышел взрывной стандарт ES6, он же называется ES2015
  • Готовится к выходу ES8
  • Как таковых ES>6 - нет, есть набор фич, которые поддерживают браузереы или нет

Что необходимо знать про JS?

  • JS - скриптовый интерпретируемый язык
  • Как только его стали минифицировать - он стал компилируемым :)
  • На ES6 пишут, а ES 5.1 только поддерживается
  • Как так?

Что необходимо знать про JS?

  • На ES6 пишут, а ES 5.1 только поддерживается
  • Процесс преобразования ES6 в ES 5.1 называет транспайлингом
  • Это осуществляется в момент сборки UI специальным утлитами/сборщиками
  • Это позволило возникнуть диалектам JS: TypeScript, Flow и другим
  • Через пару недель про это продолжим :)

Что важно помнить про CSS

  • CSS - это стили и отображение
  • Если проблему можно решить с помощью CSS, то она решается только с помощью CSS
  • Текущая версия - CSS 3, но как таковой поддержки нет - это набор фич
  • CSS - редко пишут в чистом виде
  • Stylus, Compass, Less, SASS/SCSS - это всё CSS препроцессоры
  • И, да, CSS опять, получается, компилируемый язык

Где CSS можно писать?

  • В аттрибуте style="" любого тега
  • В теге <style> страницы
  • В отдельном файле, подключив через
  • Посмотри текст этого слайда и в вверху страницы
  • Именно в этом порядке и применяются стили, если они конфликтуют

Mobile-first

  • Джедайская техника
  • Так реально проще
  • Так реально быстрее
  • 55 % всех сайтов в России открывались с мобильного (2015)
  • В остальном мире - больше

Mobile-first

  • Сначала делаем дизайн для мобильных
  • Расширяем границы - для планшетов добавляем стили
  • Для десктоп, для буржуев с большими экранами
  • По идее @media max-width у Вас не должно быть

Mobile-first (CSS)

            
.header {
    width: 130px;
    // ....
}
@media (min-height: 568px) {
    .header {
        width: 300px
    }
}
@media (min-height: 768px) {
    .header {
        font-size: 16px;
    }
}
